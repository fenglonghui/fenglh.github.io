### 定制化数据处理及向量库存储

####  结构化处理知识库数据

       对数据进行结构化处理, 回复的质量才会好

       在之前的纯RAG知识库案例中, 使用SimpleDirectoryReader粗略读区整个文档作为默认的一个节点, 进行检索查询, 这样在整个知识库中进行检索查询, 得到的结果比较模糊, 不精准, 
       要想得到精准的答复,必须对知识库数据进行结构化处理!

       结构化处理数据集:
         1.每一行作为检索查询的最小单元, 即一行为一个节点
         2.每一行显示规则: 法律名称 条款数: 条款对应的内容
         3.所有的数据集以json格式显示, 全部都放在一个列表中
           如: [{"中华人民共和国劳动法 第一条": "XXXXXXXXXXXXXXXXXXXXXXXX"}]

####  向量数据库存储
      知识库库存储选择存储在chroma向量数据库中

      安装依赖:
        pip install chromadb==1.1.1
        pip install llama-index-vector-stores-chroma
